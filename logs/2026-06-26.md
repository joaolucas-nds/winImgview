# Log — 2026-06-26

## Objetivo da sessão
Iniciar projeto de viewer de imagens para Windows com suporte a SVG vetorial e WebP. Fazer pesquisa aprofundada de ferramentas existentes antes de decidir se há necessidade de build customizado. Estruturar toda a documentação de contexto para transitar entre conversas.

## Feito
- Pesquisa aprofundada: 8 viewers Windows avaliados para suporte a SVG (vetorial vs. rasterizado), WebP, segurança, gratuidade, open source, portable.
- Identificado o problema raiz do SVG em viewers nativos: sem engine web (WebView2/Chromium), o SVG é rasterizado e pixela no zoom — explicação completa em HISTORICO.md § 1.
- Selecionada combinação primária: ImageGlass 9.5 portable (SVG vetorial + WebP + 90 formatos) + FastStone 8.5 portable (raster/RAW/batch). Justificativa em DEC-002.
- Descartados: Honeyview (descontinuado), BandiView (ad-supported), IrfanView para SVG, XnView para SVG, agregadores de download.
- Mapeadas 5 armadilhas críticas (CONTEXT.md § Armadilhas).
- Identificada stack de build condicional: Tauri 2.x + WebView2 (Fase 4 do ROADMAP, só se gap for confirmado na Fase 3).
- Capturadas 9 ideias de funcionalidades para viewer customizado (IDEAS.md).
- Documentação completa criada: CONTEXT.md, STATUS.md, DECISIONS.md, CHANGELOG.md, IDEAS.md, ROADMAP.md, GLOSSARY.md, HISTORICO.md, LOG-TEMPLATE.md, logs/2026-06-26.md.
- Três decisões registradas: DEC-001 (pesquisar antes de construir), DEC-002 (ImageGlass + FastStone), DEC-003 (fonte oficial + portable).

## Aprendizados / armadilhas
- SVG requer engine web para renderização vetorial; qualquer viewer sem engine web pixela SVG no zoom. Já registrado em CONTEXT.md § Armadilhas #2.
- MSI do ImageGlass inclui prompt Opera → usar portable ZIP do GitHub. Já em CONTEXT.md § Armadilhas #1.
- Honeyview foi descontinuado em mai/2024 → substituto BandiView é ad-supported. Já em CONTEXT.md § Armadilhas #3.
- ImageGlass 10 Beta 2 existe (jun/2026) e tem SVG nativo sem WebView2, mas é beta instável. Já em CONTEXT.md § Armadilhas #5.

## Próximos passos
1. Baixar ImageGlass 9.5 portable de `github.com/d2phap/ImageGlass/releases` (arquivo `.zip`, não `.msi`).
2. Extrair para `C:\Ferramentas\ImageGlass\` (ou pasta de preferência).
3. Configurar como programa padrão para `.svg`, `.webp`, `.png`, `.gif`, `.avif`, `.heic`.
4. Testar: abrir SVG → zoom 400% → confirmar sem pixel. Abrir WebP animado → confirmar animação.
5. Opcionalmente: baixar FastStone 8.5 portable de `faststone.org`.
6. Após 1–2 semanas de uso: avaliar se gap persiste → decisão da Fase 3 (encerrar ou build).
